# HalfTrans 产品设计文档 v1.0.0

## 1. 产品定位

**产品名称**：HalfTrans

**一句话定位**：面向互联网从业者的 Chrome 翻译插件，将英文内容转化为"最小认知成本的中英混合表达"。

**核心差异化**：不是全翻译，而是保留专业术语原文，让用户用最低的切换成本理解内容。

**目标用户**：互联网从业者（开发、产品、设计、运营等）

**目标语言**：v1.0.0 仅支持英译中

## 2. 核心功能

| 功能 | 描述 |
|------|------|
| 全文翻译 | 可视区域优先，逐段渐进式翻译，滚动时继续 |
| 选中翻译 | 选中文本后通过浮动按钮或右键触发 |
| 术语保留 | 用户自定义术语表 > AI 自动判断，全文一致性 |
| 翻译风格 | 偏口语化 / 偏书面化 |
| 保留强度 | 保守（多保留英文）/ 激进（尽量翻译） |
| 多模型支持 | 兼容任何 OpenAI API 格式的模型 |
| 快捷键 | 支持键盘快捷键触发翻译 |

## 3. 用户交互

### 3.1 触发方式

| 方式 | 场景 | 行为 |
|------|------|------|
| 右键菜单 | 全文翻译 / 选中翻译 | 右键 → "HalfTrans 翻译" |
| 浮动按钮 | 选中文本后 | 选中文字 → 文本附近出现小图标 → 点击翻译 |
| 快捷键 | 全文翻译 | 如 `Ctrl+Shift+T`（可自定义） |

### 3.2 翻译展示

- 翻译结果显示在原文段落下方
- 原文保持不动，翻译结果以视觉上可区分的样式（浅色背景/左侧边线）展示
- 每段翻译结果带一个"关闭"按钮，用户可逐段隐藏

### 3.3 全文翻译流程

1. 用户触发全文翻译
2. 检测当前可视区域内的段落
3. 逐段调用 AI 翻译，结果渐进式插入原文下方
4. 用户滚动页面时，新进入视口的段落自动触发翻译
5. 已翻译的段落不会重复翻译

### 3.4 选中翻译流程

1. 用户选中一段文本
2. 选中文本旁出现浮动翻译按钮（或右键选择）
3. 点击后翻译结果以 tooltip/浮层形式显示在选中文本附近
4. 点击页面其他区域或按 Esc 关闭

## 4. Popup 面板与设置页

### 4.1 Popup 面板

点击浏览器工具栏图标弹出：

```
┌─────────────────────────┐
│  HalfTrans              │
├─────────────────────────┤
│  [▶ 翻译当前页]         │
├─────────────────────────┤
│  翻译风格：口语化 / 书面化  │
│  保留强度：保守 / 激进      │
├─────────────────────────┤
│  术语表：已有 12 条 [编辑→] │
├─────────────────────────┤
│  ⚙ 设置                  │
└─────────────────────────┘
```

- 术语表仅显示数量和跳转入口，不展示完整列表

### 4.2 设置页（Options Page）

| 设置项 | 说明 |
|--------|------|
| API Base URL | 模型服务地址（默认 OpenAI） |
| API Key | 用户自己的密钥 |
| 模型选择 | 下拉选择或手动输入模型名 |
| 翻译风格 | 口语化 / 书面化 |
| 术语保留强度 | 保守 / 激进 |
| 自定义术语表 | 表格形式，支持增删改，可导入/导出 |
| 快捷键设置 | 自定义全文翻译快捷键 |

### 4.3 数据存储

- 所有设置和术语表存储在 `chrome.storage.sync`（跨设备同步）
- 翻译缓存使用 `chrome.storage.local`（避免同一页面重复翻译）

## 5. 技术架构

### 5.1 总体方案

纯前端架构（Manifest V3），预留后端扩展点。AI 调用抽象成统一的 `TranslationProvider` 接口，v1.0.0 实现为 `DirectAPIProvider`（用户 Key 直接调用），未来加付费服务时新增 `HostedProvider` 即可。

### 5.2 模块职责

| 模块 | 职责 |
|------|------|
| Service Worker (Background) | 管理右键菜单注册、快捷键监听、协调翻译任务 |
| Content Script | 页面 DOM 操作：检测段落、插入翻译结果、浮动按钮、视口监听 |
| Popup | 快捷操作面板 UI |
| Options Page | 完整设置页面 |

### 5.3 翻译核心抽象

```
TranslationProvider (接口)
├── DirectAPIProvider（v1.0.0：用户 Key 直接调用）
└── HostedProvider（未来：付费托管服务）
```

### 5.4 数据流

```
用户触发 → Content Script 收集文本段落
  → 发消息给 Service Worker
  → Service Worker 调用 TranslationProvider
  → TranslationProvider 带上术语表 + 风格 + 强度构造 Prompt
  → AI API 返回结果
  → Service Worker 回传给 Content Script
  → Content Script 插入翻译结果到 DOM
```

### 5.5 Prompt 设计要点

- 系统 Prompt 中注入：翻译风格、保留强度、术语表
- 全文翻译时，先扫描一遍可见段落提取出现的术语，确保一致性
- 每次请求带上已翻译段落的术语处理记录，防止前后不一致

## 6. 错误处理

### 6.1 API 相关

| 场景 | 处理方式 |
|------|----------|
| API Key 未配置 | 翻译触发时弹出提示，引导去设置页配置 |
| API 调用失败（网络错误） | 对应段落显示"翻译失败，点击重试"，不影响其他段落 |
| API 速率限制 | 自动降速，排队重试，用户可见加载状态 |
| 模型返回格式异常 | 降级展示原始返回内容，标记为"翻译结果可能不准确" |

### 6.2 页面相关

| 场景 | 处理方式 |
|------|----------|
| 页面内容动态加载（SPA） | 监听 DOM 变化（MutationObserver），新内容进入视口时自动翻译 |
| 已翻译段落原文被更新 | 检测变化，清除旧翻译，重新翻译 |
| 页面内容非英文 | 不触发翻译，或提示"未检测到英文内容" |
| 极长段落 | 按句子拆分发送，避免超出模型 Token 限制 |

### 6.3 用户操作

| 场景 | 处理方式 |
|------|----------|
| 翻译过程中用户滚动 | 当前段落继续翻译，新可见段落加入队列 |
| 翻译过程中关闭页面 | 中断请求，无副作用 |
| 重复触发全文翻译 | 跳过已翻译段落，只翻译未翻译的 |

### 7.1 技术栈

|------|------|

1. 支持多语言（英译日、英译韩等）
2. 支持不同行业术语库（金融、医疗、法律等）

4. 付费托管服务（HostedProvider + 订阅制/按量付费）
